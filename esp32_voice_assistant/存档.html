一、项目概览

一个内置 ESP32-S3 的智能交互玩偶「小小倪」，具备语音对话、长期记忆、自定义角色、语音控制设备的能力。项目初衷是异地恋期间，用倪兆贤的声音复刻和自定义角色设定，陪伴女友王彤彤度过整个暑假。

  • 硬件：ESP32-S3 (N16R8) + INMP441 麦克风 + MAX98357 功放 + 喇叭 + 按钮
  • 语音链路：按住录音 → 松手 → 百度 REST ASR → DeepSeek 流式 LLM → CosyVoice TTS → PCM 播放
  • 角色：名字「小小倪」，使用 CosyVoice 3.5 Flash 复刻声音，所有对话以倪兆贤的声音说出
  • 联网：WiFiManager 自动配网，WebSocket 流式协议，腾讯云 24h 运行
  • 暖心功能：见面倒计时、天气关怀、暖心语录池、晚安自动提醒、主动交互
  • 部署：systemd 自启 + 崩溃恢复，日志自动清理，对话历史自动裁剪

协议设计 (WebSocket 流式)

方向消息类型说明
ESP32 → 服务器二进制PCM 音频块（每 ~150ms 发一块）
ESP32 → 服务器文本 "DONE"录音结束，触发服务端处理
服务器 → ESP32二进制PCM 音频块（CosyVoice 流式输出）
服务器 → ESP32文本 "DONE"所有音频发送完毕

四、对话流程

  1. 用户按住按钮 (GPIO 0)
  2. ESP32 通过 I2S 从 INMP441 读取音频，存入缓冲区
  3. 边录边传：每 ~150ms 自动降混为单声道 PCM，通过 WebSocket 发送
  4. 用户松开按钮，发送剩余数据 + DONE
  5. 服务器：PCM→WAV → 百度 ASR → DeepSeek LLM 流式 → CosyVoice TTS → PCM 推回
  6. 服务器发送 DONE，ESP32 写 I2S → 喇叭播放

六、PlatformIO 编译适配

项目最初在 Arduino IDE（ESP32 核心 3.x，ESP-IDF 5.x 新 I2S API）下开发。云服务器 PlatformIO 使用 espressif32@6.9.0（Arduino 2.0.17，仅支持旧 I2S API），需做以下适配：

  • I2S API 从新版（i2s_new_channel、i2s_std_config_t）改写为旧版（i2s_driver_install、i2s_config_t）
  • #include <AudioOutputI2S.h> → #include <AudioOutput.h>
  • #include <driver/i2s_std.h> → #include <driver/i2s.h>
  • ESP8266Audio v2.4.1 移除 AudioOutputI2S.cpp、AudioOutputI2SNoDAC.cpp、AudioOutputPDM.cpp、AudioOutputSPDIF.cpp（项目未使用）
  • ota_update.h 中使用 esp_partition_find_first 替代 esp_ota_get_running_partition

本地 Arduino IDE 和服务器各维护一个 API 版本。本地用新 API，服务器用旧 API + PlatformIO 编译。

  • 固件存储：~/firmware/ 目录，按 firmware_v{x.x.x}.bin 命名
  • 版本清单：GET /api/firmware/manifest 返回最新版本、URL、MD5、大小
  • 当前版本：v1.1.0（编译于 2026-06-23）
  • 检查频率：ESP32 每 6 小时自动检查，或重启时检查

十四、已修复问题

1. 按钮防抖逻辑 Bug（06-17）

static bool lastBtn = HIGH; 硬编码导致虚假状态跳变。改用 lastBtn=-1 首次读取真实状态。

2. 扬声器无声音（06-18）

面包板 GND 排接触不良。整理共地后恢复。

3. Arduino IDE / PlatformIO 编译冲突（06-20~23）

两个环境的 ESP32 Arduino 核心版本不同（本地 3.x vs 服务器 2.0.17），I2S API 不兼容。最终维护两套 API 版本，服务器用旧 API + PlatformIO 6.9.0 编译。

4. WebSocket keepalive ping failed（06-20）

websockets 库版本兼容问题，升级后解决。

5. 管理面板 JS 引号冲突（06-23）

onkeydown 中的 'Enter' 打断了外层单引号 JS 字符串，导致所有按钮失效。修复后硬刷新可用。
